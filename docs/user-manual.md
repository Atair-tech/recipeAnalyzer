# Recipe Analyzer 用户手册

## 1. 这是什么

`Recipe Analyzer` 是一个本地运行的菜谱知识库应用。  
它会把你维护在 `Excel` 工作簿里的菜谱，同步到本地 `SQLite` 数据库中，再提供：

- 浏览
- 筛选
- 搜索
- 数据分析
- 本地 AI 问答
- 自动标签
- 结构化食材精校

当前原则：

- `Excel` 仍然是主数据源
- Web 端不是完整编辑器
- 导入采用差异同步，不是简单追加

## 2. 如何打开

如果前后端已经启动，可直接访问：

- 前端：`http://127.0.0.1:5173`
- 后端：`http://127.0.0.1:8000`

## 3. 页面说明

### 3.1 总览

显示当前系统的核心汇总信息，例如：

- 正式菜谱数量
- 待办项数量
- 专题库数量
- 结构化食材数量
- 同步批次数量

### 3.2 菜谱库

这是日常使用最频繁的页面。

支持：

- 关键词搜索
- 按类型筛选
- 按专题库筛选
- 按分组筛选
- 按菜系筛选
- 按结构化食材筛选
- 按自动标签筛选
- 按 `BMD / CC` 筛选
- 导出当前筛选结果

详情页会显示：

- 专题库
- 分组
- 菜系 / 亚菜系
- 来源或修订备注
- 食材
- 调料
- 结构化食材
- 做法及要点
- 自动标签

### 3.3 智能问答

智能问答页面支持本地 `Ollama` 模型问答。

支持：

- 选择模型
- 是否把当前菜谱作为上下文
- 是否显示推理过程
- 流式显示阶段状态

阶段状态包括：

- `快速回复中`
- `正在解释问题`
- `正在检索菜谱`
- `正在生成回答`

说明：

- 简单短句例如 `你好 / 谢谢 / 收到 / 好的` 会走快速路径
- 复杂问题会走“解释问题 -> 检索 -> 回答”的完整流程

### 3.4 数据分析

数据分析页面提供仪表盘式统计。

支持：

- 切换统计维度
- 切换统计范围
- 切换显示条数

目前可查看的统计包括：

- 专题库分布
- 分组分布
- 菜系分布
- 亚菜系分布
- 记录类型分布
- 自动标签分布
- 食材分布

### 3.5 管理

管理区包含以下子页面：

- `标签管理`
- `导入 Excel`
- `食材审查`
- `菜谱配对`
- `查看数据库`
- `AI 对话记录`

## 4. 导入 Excel

### 4.1 基本流程

1. 修改 `data/recipes.xlsx`
2. 进入 `管理 -> 导入 Excel`
3. 预览解析结果
4. 执行同步

### 4.2 同步规则

系统会把 Excel 与当前数据库做差异比较，结果分为：

- 新增
- 更新
- 删除
- 未变化

这意味着：

- 同一份工作簿重复导入，不会无限叠加重复菜谱
- 没变化的记录不会重复写入
- Excel 中删除的记录，也会从数据库中删除

## 5. 自动标签

系统内置一套独立于 Excel 的自动标签体系。

特点：

- 标签定义保存在数据库自带表中
- 不受 Excel 导入覆盖影响
- 可由本地 AI 批量打标签
- 已处理且未变化的 recipe 会自动跳过
- 支持查看某个标签命中了哪些菜谱
- 支持手动移除错误的标签关联

## 6. AI 精校

`管理 -> 导入 Excel` 页面支持 AI 精校。

当前版本的精校范围是：

- **只精校结构化食材**

不会改写：

- `ingredients_text`
- `seasonings_text`
- `steps_text`
- `notes_text`

支持：

- 启动
- 暂停
- 恢复
- 增量跳过

增量判断依据：

- `source_hash`
- 当前模型
- `refine_version`

## 7. 食材审查

`管理 -> 食材审查` 用来审核精校结果。

可以查看：

- 原始食材文本
- 精校前结构化食材
- 精校后结构化食材
- 当前结构化食材
- 最近精校状态
- 最近错误信息

可以执行：

- 标记 `通过`
- 标记 `有问题`
- 保存备注
- 单条重跑精校

注意：

- 精校前后快照只会对新增精校或重跑后的记录开始保存
- 老的历史记录可能没有快照

## 8. 菜谱配对

`管理 -> 菜谱配对` 用于处理：

- 仅索引页
- 仅做法页
- 疑似未正确配对的条目

可以：

- 查看系统建议
- 手动指定配对
- 保存配对规则

## 9. 查看数据库

`管理 -> 查看数据库` 页面提供数据库表格浏览功能。

支持：

- 查看主要业务表
- 浏览表数据
- 导出数据库
- 导入数据库

说明：

- 导出会下载当前 `SQLite` 数据库文件
- 导入会替换当前数据库
- 导入前建议先备份

## 10. AI 对话记录

`管理 -> AI 对话记录` 用于查看本地 AI 调用记录。

支持查看：

- 功能类型
- 阶段
- 模型名
- 请求消息
- 模型回复
- 错误信息

适合排查：

- 智能问答
- 自动标签
- AI 精校

## 11. 独立精校工具

项目还提供一个独立工具：

- `desktop/db_refiner/dist/DataHelper.exe`

用途：

- 在另一台安装了 `Ollama` 和本地模型的电脑上
- 直接打开导出的数据库文件
- 对数据库中的菜谱进行结构化食材 AI 精校

说明：

- 它会直接写回你打开的那个数据库文件
- 支持暂停和恢复
- 运行结束会显示：
  - 进度
  - 成功数
  - 跳过数
  - 错误数
  - 首条错误

## 12. 使用注意事项

- Excel 仍然是主数据源
- 自动标签和结构化食材精校都依赖本地模型质量
- 本地模型运行速度受硬件影响很大
- 大模型全量精校可能持续数小时甚至更久
- 如果数据库是在别的机器上被独立工具修改过，需要重新导入或替换数据库文件
