# Recipe Analyzer 用户手册

## 1. 这是什么

`Recipe Analyzer` 是一个本地运行的菜谱库应用。它会把 Excel 工作簿中的菜谱同步到本地 SQLite 数据库，并提供浏览、筛选、搜索、数据分析、AI 辅助整理和数据库管理功能。

基本原则：

- Excel 是主数据源。
- SQLite 是程序运行时使用的本地数据库。
- Web 端主要用于查看、筛选、分析和管理，不是完整编辑器。
- AI 生成内容只作为辅助参考，原始 Excel 内容始终优先。

## 2. 如何打开

如果前后端已经启动，可访问：

- 前端：`http://127.0.0.1:5173`
- 后端：`http://127.0.0.1:8000`

第一次打开会显示启动门动画。点击进入后，程序会在浏览器中记住这个状态，后续刷新页面不会再次显示启动门。

## 3. 首次使用流程

1. 确认后端和前端已经启动。
2. 进入 `管理 -> 导入 Excel`。
3. 选择 `data/recipes.xlsx`。
4. 点击预览，确认解析结果。
5. 点击同步到 SQLite。
6. 进入 `菜谱库` 浏览菜谱。
7. 可选：进入 `管理 -> AI分析食材` 执行 AI 食材分析。
8. 可选：进入 `管理 -> 标签管理` 执行自动标签。
9. 如需备份，进入 `管理 -> 查看数据库` 导出数据库。

## 4. 页面说明

### 4.1 总览

显示当前菜谱库的统计信息，例如：

- 正式菜谱数量。
- 待办项数量。
- 专题库数量。
- 有做法记录数量。
- 可见标准化食材数量。
- 同步批次数量。

统计维度包括：

- 专题库分布。
- 分组分布。
- 菜系分布。
- 亚菜系分布。
- 记录类型分布。
- 自动标签分布。
- 食材分布。

食材分布只统计已通过显示清理的可见标准化食材。

### 4.2 菜谱库

这是日常使用的核心页面。

支持：

- 关键词搜索。
- 按类型筛选。
- 按专题库筛选。
- 按分组筛选。
- 按菜系筛选。
- 按可见标准化食材筛选。
- 按自动标签筛选。
- 按 `BMD / CC` 筛选。
- 导出当前筛选结果。

详情页会显示：

- 专题库、分组、菜系、亚菜系。
- 最后记录日期。
- 来源或修订备注。
- 食材原文。
- 调料原文。
- 做法及要点原文。
- 标准化食材（AI生成，仅供参考）。
- 自动标签（AI生成，仅供参考）。
- 原始 Excel 记录。

注意：

- “食材”“调料”“做法及要点”来自原始 Excel，是最可靠内容。
- “标准化食材”和“自动标签”由 AI 或规则生成，仅用于筛选、统计和推荐参考。

### 4.3 智能问答

智能问答包含两个子模块：

- `AI问答`
- `自然语言搜索`

AI问答支持：

- 选择本地 Ollama 模型。
- 显示或隐藏思考过程。
- 可选是否使用当前菜谱作为上下文。
- 可选 DeepSeek 前置解析。
- 可选 DeepSeek 后处理。
- 查看本轮检索和引用。
- 停止正在生成的回答。
- 清空对话。

说明：

- 默认不把当前菜谱作为上下文。
- 默认不启用 DeepSeek 后处理。
- 如果未配置 DeepSeek API Key，DeepSeek 选项会明确显示未配置 Key，未输入 Key 前不会启动外部调用。
- DeepSeek 前置解析只发送用户问题和低风险库内词表摘要。
- DeepSeek 后处理会发送本地候选菜谱的脱敏摘要，可能包含菜名、分类、自动标签、简短食材或调料摘要和本地命中原因，但不发送完整做法、备注或完整数据库。
- 外部 API 可能产生费用。

自然语言搜索用于不调用问答模型的检索，例如：

- “不辣的牛肉饭”
- “适合下饭的鸡肉菜”
- “清淡一点的汤”

### 4.4 管理

管理菜单默认收起，包含：

- 标签管理。
- 导入 Excel。
- AI分析食材。
- 菜谱配对。
- 查看数据库。
- AI 对话记录。
- 食材审查。

## 5. 导入 Excel

入口：`管理 -> 导入 Excel`

基本流程：

1. 修改 Excel 源文件。
2. 在页面中选择 Excel。
3. 点击预览结构。
4. 检查解析结果。
5. 点击同步到 SQLite。

同步规则：

- 新增：Excel 中有、数据库中没有。
- 更新：Excel 中有变化。
- 删除：Excel 中删除的记录会从数据库中删除。
- 未变化：不会重复写入。

这套机制可以避免每次导入都全量覆盖已有 AI 分析结果。

## 6. AI分析食材

入口：`管理 -> AI分析食材`

该页面包含两步。

### 6.1 step1 本地AI分析食材

用途：

- 调用本地 Ollama 模型。
- 对每道菜的原始食材文本做结构化分析。
- 生成 `recipe_ingredients`。
- 不改写原始 Excel 文本。

增量逻辑基于：

- `source_hash`
- `model`
- `refine_version`

如果菜谱原文和模型版本没有变化，已处理记录会自动跳过。

### 6.2 step2 外部AI剔除杂乱项

用途：

- 对全局标准化食材候选词做显示清理。
- 默认使用 DeepSeek API。
- 只发送食材候选词本身。
- 不发送菜名、做法、备注或完整菜谱内容。

结果：

- 被保留的食材会出现在菜谱库食材筛选和总览统计中。
- 被剔除的杂乱项不会显示和统计。
- 原始数据库记录不会删除，便于后续继续精校。

DeepSeek Key：

- 如果未配置 Key，点击启动时会弹出输入框。
- 未输入 Key 前不会启动任务。
- Key 会保存到本机项目根目录 `.env`，不会提交到 GitHub。

## 7. 长任务说明

以下任务会在后端后台运行：

- step1 本地AI分析食材。
- step2 外部AI剔除杂乱项。
- 自动标签任务。

运行期间：

- 可以切换到其他页面。
- 关闭浏览器页面通常不影响任务。
- 只要后端服务不关闭，任务会继续运行。
- 如果关闭后端窗口，任务会中断。
- 后端重启时，遗留的 `running` 任务会自动标记为 `paused`。
- 任务变为暂停后，可以在对应页面点击“恢复”继续。

## 8. 自动标签

入口：`管理 -> 标签管理`

功能：

- 新增标签。
- 编辑标签。
- 停用或删除标签。
- 启动本地 AI 自动打标签。
- 暂停和恢复打标签任务。
- 查看某个标签命中了哪些菜谱。
- 手动移除错误标签关联。

注意：

- 标签定义保存在数据库自带表中，不受 Excel 导入覆盖。
- 自动标签由 AI 生成，仅供筛选、统计和推荐参考。
- 如果标签结果不准确，应在标签管理中审查和移除。

## 9. 食材审查

入口：`管理 -> 食材审查`

用途：

- 查看原始食材文本。
- 查看精校前结构化食材快照。
- 查看精校后结构化食材快照。
- 查看当前结构化食材。
- 查看最近精校状态和错误信息。
- 标记通过或有问题。
- 保存审查备注。
- 单条重跑精校。

说明：

- 快照只对新增精校或重跑后的记录开始积累。
- 历史旧记录可能没有快照。
- 如果 AI 结果明显错误，应以 Excel 原文为准，并通过重跑或人工审查处理。

## 10. 菜谱配对

入口：`管理 -> 菜谱配对`

用于处理：

- 仅索引页存在的条目。
- 仅做法页存在的条目。
- 疑似未正确配对的条目。

支持：

- 查看系统建议。
- 手动指定配对。
- 保存配对规则。

## 11. 查看数据库

入口：`管理 -> 查看数据库`

功能：

- 浏览数据库表。
- 查看表结构和部分数据。
- 导出当前 SQLite 数据库。
- 导入 SQLite 数据库。

说明：

- 导出的是当前数据库文件。
- 导入会替换当前数据库。
- 导入前程序会自动备份现有数据库到 `data/backups/`。
- 重要数据仍建议手动保留备份。

## 12. AI 对话记录

入口：`管理 -> AI 对话记录`

用于排查：

- 智能问答。
- DeepSeek 前置解析。
- DeepSeek 后处理。
- 自动标签。
- 食材分析。

可查看：

- 功能类型。
- 调用阶段。
- 模型名称。
- 请求内容。
- 模型回复。
- 错误信息。

`desktop/db_refiner/dist/DataHelper.exe`

## 14. 常见问题

### 14.1 Ollama 未连接

检查：

- Ollama 是否已经启动。
- 浏览器或命令行是否能访问 `http://127.0.0.1:11434/api/tags`。
- 需要的模型是否已经安装，例如 `qwen3:4b`。

### 14.2 DeepSeek 未配置 Key

表现：

- DeepSeek 相关选项显示未配置 Key。
- step2 启动时弹出输入框。
- 未输入 Key 前不会启动任务。

处理：

- 在弹窗中输入 API Key。
- 或在 `.env` 中配置 `RECIPE_ANALYZER_DEEPSEEK_API_KEY`。

### 14.3 AI 任务很慢

原因可能包括：

- 本地模型较大。
- 显卡显存不足。
- 上下文较长。
- 任务数量多。
- 外部 API 正在等待完整结果。

处理：

- 可以让任务继续后台运行。
- 可以暂停后稍后恢复。
- 可以使用独立工具在另一台电脑处理数据库。

### 14.4 任务显示 running 但没有计算资源占用

如果后端曾在任务运行中关闭，旧任务可能残留为 running。当前版本会在后端启动时自动将这类任务标记为 paused，然后可以点击恢复。

### 14.5 AI 结果和 Excel 原文不一致

以 Excel 原文为准。AI 生成的标准化食材和自动标签只用于辅助筛选、统计和推荐。

## 15. 交付包建议

第一版交付时建议包含：

- 当前数据库 `.db` 文件。
- 当前 Excel 源文件。
- `README.md`。
- `docs/user-manual.md`。
- `.env.example`。
- `DataHelper.exe`。
- `LabelHelper.exe`。
- 启动说明。

不要只交 GitHub 仓库，因为仓库通常不会包含真实数据库、Excel 文件、前端构建产物和本地 `.env`。

## 16. 当前边界

- Web 端不是主编辑器，主数据仍在 Excel 中维护。
- 自动标签和结构化食材依赖模型能力，仍需要审查。
- 本地小模型对复杂需求的理解能力有限。
- DeepSeek 可以提高复杂语义能力，但需要网络、API Key 和费用，并涉及隐私取舍。
- 智能问答不能替代医学、营养或食品安全判断。
